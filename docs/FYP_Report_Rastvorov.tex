\chapter*{Abstract}
This thesis presents the continued development of the English Language Assistant (ELA), a full-stack language learning system designed to make authentic English input more accessible without simplifying or rewriting the source material. ELA ingests text, audio, and video, then transforms the input into a hierarchical contract of \emph{Recursive Linguistic Elements} (RLE), where each sentence is represented as a structured tree of sentence, phrase, and word nodes enriched with grammatical, pedagogical, and bilingual metadata.

The implemented system combines deterministic linguistic processing and machine learning in a rules-first architecture. spaCy-based parsing, a deterministic skeleton builder, and a rule-based tense-aspect-mood layer provide the structural truth layer. Additional enrichments include CEFR estimation, pedagogical note generation, translation, phonetic transcription, vocabulary export, and runtime persistence. A React/Vite progressive web application and a runtime service expose these results through an interactive visualizer and media-oriented workflow.

Alongside the engineering contribution, the project examines a central modelling insight that emerged during note-generation experiments: performance improved substantially when raw lexical notation was replaced by structurally abstract placeholder templates. This observation is interpreted as a \emph{context--pattern duality}: during training, diverse contextual examples carve stable internal patterns; during inference, frozen patterns impose structure on unseen context. In this thesis, that idea is treated as an empirical observation and explanatory framework rather than a formal theorem.

The final outcome is not only a prototype application but a validated architectural framework for explainable, AI-assisted language learning. At the same time, the thesis documents current limitations, including unresolved runtime and pipeline regressions, incomplete stabilization of some multimodal paths, and the gap between research claims and fully production-hardened behavior. The work therefore positions ELA as both a usable system and an ongoing research programme.

\chapter{Introduction}
\section{Motivation}
Most digital language-learning systems lower difficulty by reducing or simplifying the source material. This makes early progress easier, but it also distances the learner from real English as it appears in speech, media, and written communication. Authentic input is richer, more motivating, and more representative of actual language use, but it is also harder to process without support. The central motivation of ELA is therefore not to simplify authentic English, but to make it explainable.

\section{Problem Statement}
The problem addressed in this project is the lack of a transparent, learner-facing system that can process authentic English across text, audio, and video, preserve the original linguistic structure, and expose just enough grammatical and bilingual support to improve comprehension. Existing systems typically optimize for fluency drills, vocabulary repetition, or black-box assistance. ELA instead targets explainable decomposition of authentic input into interpretable linguistic units.

In scope for this thesis are the implemented architecture, the linguistic contract, the multimodal runtime path, the frontend-facing visualization flow, and the experimental line around note-generation abstraction. Out of scope are large-scale classroom deployment and strong claims of measured learning gain through user studies. This boundary is intentional: the thesis evaluates a research-engineering system, not a fully validated educational product.

\section{Research Questions}
This thesis is guided by three research questions:
\begin{enumerate}
  \item How can a rules-first linguistic pipeline produce structured, pedagogically useful representations from authentic media without simplifying the original input?
  \item What representation model best captures sentence, phrase, and word relationships while remaining stable enough for validation, persistence, and visualization?
  \item To what extent can structural abstraction, particularly placeholder-based modelling, improve the generalization of generated pedagogical notes?
\end{enumerate}

\section{Objectives}
The implementation phase of ELA is organized around the following objectives:
\begin{enumerate}
  \item Build a multimodal processing pipeline that ingests text, audio, and video and converts them into structured sentence-level representations.
  \item Formalize a hierarchical contract that can carry grammatical, pedagogical, and bilingual metadata in a stable and validated form.
  \item Deliver a learner-facing application that exposes linguistic structure through an interactive visualizer, export flow, and media-oriented workflow.
  \item Investigate whether abstract structural templates support stronger note-generation generalization than raw lexical notation.
\end{enumerate}

\section{Contributions}
The main contributions of this thesis are:
\begin{itemize}
  \item an evolved implementation of ELA as a full-stack system rather than a concept-level prototype;
  \item the formalization and operationalization of the RLE contract as the core data interface between backend, storage, and frontend;
  \item a rules-first linguistic pipeline integrating parsing, phrase construction, TAM enrichment, CEFR estimation, translation, phonetic support, and pedagogical notes;
  \item a research interpretation of the placeholder strategy for note generation through the context--pattern duality;
  \item a critical evaluation of the current system, including implemented capabilities, empirical evidence, and unresolved limitations.
\end{itemize}

Taken together, these contributions position the thesis as both an implementation report and an argument about representational design. The software contribution is the operational ELA stack itself. The research contribution is the claim that stronger pedagogical behaviour emerged when linguistic variation was reorganized into more stable structural forms.

\section{Thesis Structure}
The remainder of this report is structured as follows. Chapter 2 reviews the background fields relevant to ELA. Chapter 3 positions the current work as a continuation of the earlier research thesis and explains the project evolution. Chapter 4 defines the RLE data model. Chapter 5 examines the context--pattern duality. Chapter 6 presents the implemented system architecture. Chapter 7 discusses dataset and corpus engineering. Chapter 8 evaluates the system. Chapter 9 discusses limitations and future work. Chapter 10 concludes the thesis.

\chapter{Background and Related Work}
\section{Computer-Assisted Language Learning}
Computer-Assisted Language Learning (CALL) provides the pedagogical backdrop of ELA. Classical CALL systems focused on drills, exercises, and tightly constrained curricula [4], while later work emphasized broader software-mediated support for second-language acquisition [5]. More recent systems incorporate adaptation, learner modelling, multimodal content, and mobile delivery. Even so, a large portion of practical CALL still prioritizes task completion and fluency support over structural transparency. The learner receives an answer, a translation, or a corrected sentence, but not necessarily an interpretable representation of why the form is what it is.

ELA aligns with a narrower strand of CALL that values learner autonomy, authentic materials, and graduated support. In that tradition, software should not simply replace the learner's analysis; it should externalize part of the analysis in a form that the learner can inspect. This matters especially in independent learning scenarios, where the system must function not only as an exercise engine but as a structured explanatory companion.

\section{Authentic Materials and Cognitive Support}
Authentic materials are attractive because they preserve the lexical, syntactic, pragmatic, and stylistic richness of real communication. This emphasis is well established in language-teaching literature on authenticity and materials design [1]--[3]. They expose the learner to naturally occurring tense shifts, ellipsis, collocation, discourse markers, and speaker intent in a way that sanitized textbook examples often do not. At the same time, the very properties that make authentic input valuable also make it cognitively demanding.

This creates a design tension. If the material is simplified too aggressively, authenticity is lost. If it is left untouched without support, the learner can become overloaded. ELA addresses this gap by layering support over the original input rather than rewriting it. The project therefore sits at the intersection of authenticity, scaffolding, and transparency: it tries to preserve the original utterance while exposing interpretable structure around it.

\section{NLP Parsing and Structured Analysis}
The system relies on modern NLP tooling to obtain syntactic and morphological signals, especially tokenization, dependency parsing, lemmatization, and part-of-speech tagging. These signals are useful, but they are not themselves a learner-facing representation. Raw parser output is often unstable, implementation-specific, and difficult to expose directly in a pedagogical interface.

ELA therefore treats parsing as an evidence source rather than as the final user model. Parser outputs are normalized into a controlled hierarchical representation that can support enrichment, validation, storage, and visualization. This distinction is important: the project is not merely a wrapper over an NLP library, but an attempt to convert low-level linguistic signals into a stable pedagogical contract.

\section{Tense, Aspect, and Mood}
Verb morphology is central to pedagogical explanation in ELA because tense, aspect, and mood are among the most common sources of learner confusion. In many learning tools, TAM interpretation is either hidden inside a black-box model or reduced to surface labels that are difficult to audit. ELA adopts a more conservative approach. The implemented system uses a deterministic rules layer over parser output to infer TAM-related information and to preserve traceable links between grammatical evidence and pedagogical phrasing.

This is not only an engineering convenience. It is also a methodological choice. A deterministic TAM layer makes failure analysis easier, allows targeted tests, and prevents downstream note generation from becoming the sole source of grammatical truth.

\section{CEFR Estimation}
The Common European Framework of Reference for Languages offers a widely recognized scale for expressing learner difficulty and progression. In ELA, CEFR is not treated as a complete pedagogical theory, but as a practical enrichment layer that helps characterize sentence and node difficulty in a form familiar to teachers and learners.

The repository shows that multiple approaches were explored, including classifier-driven and tabular strategies. The implemented system converges toward a lighter-weight deployment path because it is easier to integrate into a contract-oriented architecture and easier to reason about operationally. This reflects a broader pattern in the project: simpler, auditable components are often preferred when they fit the explanatory goals better than heavier black-box alternatives.

\section{Sequence-to-Sequence Models and T5}
Sequence-to-sequence models are relevant because pedagogical note generation in ELA is not a pure classification problem. It requires controlled wording generation from structured inputs. T5-style models are a natural fit for this kind of mapping, especially when the input can be expressed as a text-like notation carrying grammatical state [7].

However, the project evidence suggests that representation strategy matters at least as much as model choice. The strongest note-generation behavior did not come simply from scaling model size or training longer. Instead, it emerged when the training signal was restructured through placeholders and canonicalized patterns. This observation motivates the later discussion of context--pattern duality.

\section{Automatic Speech Recognition and Translation}
Authentic language learning is not restricted to text. Spoken input introduces pronunciation, pacing, disfluency, and subtitle alignment problems that do not appear in static text. ELA therefore extends to audio and video through ASR, sentence segmentation, subtitle generation, and translation enrichment.

Whisper-based transcription and M2M100-based translation represent the local model path visible in the codebase and documentation, while additional providers remain optional or comparative extensions [8], [9]. This multimodal dimension makes ELA more ambitious than a text-only parser, but it also increases integration fragility because sentence boundaries, timestamps, and translated artifacts must remain synchronized across several layers.

\section{Explainability as a Systems Property}
An important implication of the preceding literature is that explainability in language-learning software cannot be reduced to a single model output. A system becomes explainable when its internal representation, validation boundaries, interface structure, and learner-facing presentation all support inspection. This is why ELA invests so heavily in an intermediate contract. The RLE structure is not just a storage convenience; it is the mechanism through which parser evidence, grammatical interpretation, bilingual support, and frontend visualization remain aligned.

\section{Gap Analysis}
The main gap targeted by ELA is the absence of a single learner-facing platform that takes authentic multimodal English input and converts it into a structured, hierarchical, explainable, and exportable pedagogical representation. Existing tools may offer translation, subtitles, grammar help, or media playback, but ELA's distinguishing aim is the combination of:
\begin{itemize}
  \item authentic media ingestion;
  \item hierarchical structural representation;
  \item explicit validation of the intermediate contract;
  \item learner-facing visualization and export;
  \item a research path toward abstraction-based pedagogical generation.
\end{itemize}

\chapter{Previous Research Foundation and Project Evolution}
\section{Relationship to the Earlier Research Thesis}
This thesis is not the first formal document produced around ELA. A previous research thesis established the conceptual foundations of the project: the rationale for explainability over simplification, the importance of authentic input, the Recursive Linguistic Elements idea, and the pedagogical role of an interactive visualizer. That earlier work was primarily concerned with framing the problem, defining the object model, and demonstrating a prototype direction.

The current thesis continues from that base. Its focus is no longer the justification of ELA as an idea, but the implementation, validation, and critical assessment of ELA as an engineering system. This distinction matters: the current contribution should be evaluated on what was operationalized, tested, stabilized, and discovered during implementation.

\section{From Conceptual Prototype to Implemented System}
The project evolved along three main axes:
\begin{enumerate}
  \item \textbf{Architectural maturation.} Earlier design descriptions emphasized a concept-driven prototype and a more speculative integration of external AI services. The current implementation shifts the core truth layer toward deterministic parsing, rules, and validation, with generated content treated as an enrichment path rather than the foundation of correctness.
  \item \textbf{Expansion of the data contract.} The RLE concept was converted into a more formal and stricter contract that now includes node identifiers, parent relationships, source spans, dependency information, trace metadata, translations, and multiple enrichments.
  \item \textbf{Operational workflow.} The project now includes a runtime service, client persistence, media pipeline, frontend routes, export paths, and a large test surface. This moves ELA closer to a usable software product while also exposing implementation weaknesses that a purely conceptual report could avoid.
\end{enumerate}

\section{Practical Evolution Through Repository Milestones}
The repository history makes this evolution unusually visible because design decisions were documented and then implemented in clearly traceable stages. The earliest implementation framing emphasized a deterministic foundation first. The implementation proposal for the linguistic notes pipeline defined the initial practical gap very explicitly: before any serious model path could be trusted, the system needed a deterministic core consisting of parser-driven skeleton construction, TAM rules, strict validation, and a production-safe inference runner. In other words, the first major implementation move after the research thesis was not toward a larger model, but toward a safer and more reproducible structural baseline.

This foundation was then pushed into a stronger contract form. The git history shows a sequence of changes that tightened the sentence contract, unified note context flow, stabilized grammar blueprints, and eventually migrated the system to a recursive node hierarchy. That progression matters for the thesis because it marks the point where RLE stopped being mainly a conceptual representation and became an operational interface shared by inference, validation, runtime payloads, and the frontend visualizer. The curator progress report later describes this stage as a stabilization of a contract-compliant, structurally cleaner baseline with stricter validation and wider test coverage.

The next major shift was architectural rather than purely linguistic: the project moved toward a client-first product model. The product specification dated 2026-02-17 defines this stage clearly. Heavy interactive work, local projects, media handling, and learner edits were pushed toward the client, while the backend was narrowed to minimal orchestration and sentence-contract support. This decision then appears in implementation commits adding local SQLite persistence, runtime capability flags, media routing policy, frontend-ready runtime media service, project-scoped upload and processing paths, and eventually a React frontend slice with visualizer and media workflow support. As a result, the project moved from ``pipeline plus prototype UI'' toward a coherent application architecture with explicit offline/online behavior.

At the same time, the modelling strategy continued to evolve. February and March commits show the emergence of a classifier-oriented pedagogy stack: curriculum specifications, classifier orchestration, blueprint traceability checks, tabular CEFR baselines, and full-ladder evaluation checkpoints. This phase did not eliminate generation, but it changed its role. By March 2026, the technical specification no longer described T5 as a component that could freely produce pedagogical structure. Instead, the target architecture became classifier-first and contract-safe: deterministic parsing and rules own structure, tabular ML owns CEFR, and T5 is confined to controlled lexical realization from a deterministic template.

The intermediate artifacts make this transition especially clear. A report from 2026-02-13 on the template-id experiment shows that the pipeline wiring was already deterministic, but that the dataset had collapsed to only ten active template IDs after balancing and still produced a practical fallback rate of 1.0 on the small regression suite. That is an important intermediate result: the project learned that structural control alone was not enough if the target space became too compressed. A later classifier execution checkpoint from 2026-02-25 shows the opposite kind of limitation. The orchestrator, quality gates, and controlled rewrite path were all operational, but the baseline freeze still used only four training rows and two development rows. In other words, by late February the architecture was becoming coherent faster than the data scale.

That transition is one of the clearest practical bridges from the earlier research thesis to the final implementation state. In the earlier thesis, the custom model path existed mainly as a promising future direction. In the implemented repository, that direction was tested, constrained, partially validated, and reorganized. The important result is not just ``T5 was used.'' The more important result is that repeated practical work led to a narrower and more defensible design: blueprint preservation, field ownership, template expansion, controlled rendering, explicit post-render validation, and a willingness to treat weak intermediate results as design evidence rather than as something to hide.

The final major practical axis was multimodal and operational integration. The history includes media upload and extraction paths, ASR-first local processing, translated media artifact generation, translation overlays, phonetic enrichment, synonym enrichment, Docker deployment notes, and PWA integration checks. These changes are not peripheral. They show how the original educational vision was extended from text analysis to a broader learner workflow involving files, projects, media, review, persistence, and export. They also explain why the current thesis must discuss regressions and integration mismatches: once the project became an end-to-end system, weaknesses at runtime boundaries became part of the real research record rather than hidden implementation detail.

\section{Research Continuity}
Despite the architectural shift, the underlying continuity is strong. Both the earlier thesis and the current project maintain the same central educational premise: authentic English should remain intact, and support should be attached through structured interpretation rather than through simplification. The implementation phase did not replace that vision; it forced it into a more disciplined computational form.

From the perspective of the final thesis, the main risks are not only technical but evidential:

Recognizing these risks improves the quality of the report. It encourages careful wording, checkpoint-specific evidence, and explicit distinction between implemented baseline, experimental branch, and future direction.

\chapter{The RLE Data Model}
\section{Design Rationale}
Recursive Linguistic Elements (RLE) form the central data contract of ELA. The purpose of RLE is to translate parser output and downstream enrichments into a single representation that is stable enough for storage and validation, expressive enough for machine processing, and simple enough to be rendered in a learner-facing visualizer.

\section{Hierarchy}
The implemented hierarchy uses three conceptual layers:
\begin{enumerate}
  \item \textbf{Sentence} as the root pedagogical and analytical unit;
  \item \textbf{Phrase} as an intermediate structure carrying meaningful chunks such as noun phrases, prepositional phrases, or other grouped spans;
  \item \textbf{Word} as the terminal lexical unit carrying finer grammatical metadata.
\end{enumerate}

This structure is recursive because every non-terminal node contains a \texttt{linguistic\_elements} array of child nodes. That makes the contract easy to traverse, validate, render, and export.

\section{Core Fields}
The contract uses a mixture of required and optional fields. The exact field set evolved during implementation, but the most important fields include:
\begin{itemize}
  \item \texttt{type}, \texttt{content}, and \texttt{part\_of\_speech};
  \item \texttt{tense}, \texttt{aspect}, \texttt{mood}, \texttt{voice}, and \texttt{finiteness};
  \item \texttt{node\_id}, \texttt{parent\_id}, and \texttt{source\_span};
  \item \texttt{grammatical\_role}, \texttt{dep\_label}, and \texttt{head\_id};
  \item \texttt{linguistic\_notes}, \texttt{notes}, and note provenance-related fields;
  \item \texttt{translations}, \texttt{phonetic}, \texttt{synonyms}, and \texttt{cefr\_level};
  \item \texttt{schema\_version} and quality-control trace fields.
\end{itemize}

\section{Validation Modes}
The implementation retains two validation modes:
\begin{itemize}
  \item \textbf{v1}, intended as the backward-compatible mode;
  \item \textbf{v2\_strict}, intended as the current production-facing mode with stronger field requirements.
\end{itemize}

This dual-mode design is useful in theory because it enables controlled migration. In practice, the implementation review shows that validation compatibility is still imperfect, especially around null handling in TAM fields and around runtime payload expectations.

\section{How the Contract Evolved in Practice}
The repository history shows that the RLE contract did not appear fully formed. It was expanded through a series of practical corrections driven by downstream needs. Early contract work focused on making the pipeline valid and testable at all. Later commits then added normalized morphology features, dependency linkage fields, grammatical roles, node metadata, trace fields, strict validation mode, modal-perfect policy enforcement, translation validation, and subtree-level backoff diagnostics.

This evolution matters because each addition corresponds to a concrete pressure from the rest of the system. Node identifiers and parent links were needed for deterministic graph identity and frontend selection. Source spans were needed for alignment, translation projection, and edit targeting. Grammatical roles and dependency fields were needed so that the tree was not merely a nested display object, but a linguistically meaningful structure. Trace fields and rejected-candidate statistics were needed because once the project started filtering, retrying, and validating generated notes, the contract also had to record why certain outputs were accepted or rejected.

The migration to recursive hierarchy is especially important in this story. The git history shows an explicit transition away from legacy flat rules toward the recursive node contract. That move made the visualizer, runtime payloads, validator logic, and later controlled note-generation path all speak the same structural language. It also made the contract more demanding: once recursive identity and field ownership became central, schema drift and null-handling issues became more visible rather than easier to ignore.

\section{What the Canonical Sample Shows}
The repository's canonical sample contract makes several implementation decisions explicit. A sentence node is not just a container for text. It is a convergence point for structural, pedagogical, bilingual, and traceability layers:
\begin{itemize}
  \item sentence and phrase nodes can carry TAM values and pedagogical notes;
  \item node-level translations and phonetic values are attached as first-class fields rather than external lookups;
  \item \texttt{grammar\_classes}, \texttt{note\_blueprints}, and \texttt{note\_generator\_version} preserve the distinction between structural evidence and rendered learner-facing wording;
  \item \texttt{source\_span}, \texttt{node\_id}, and \texttt{parent\_id} are intended to make every node referentially stable enough for UI selection, validation, and later editing.
\end{itemize}

The field-ownership matrix dated 2026-03-04 clarifies why this sample matters architecturally. The contract is not meant to be a place where one model predicts everything. Instead, different field families belong to different sources of truth: parser and builder rules own structure, rules-first logic owns TAM and grammatical interpretation, tabular ML owns CEFR, and T5 is constrained to learner-facing note wording only. This is one of the most important implementation outcomes of the whole project, because it turns the contract from a passive schema into an active boundary between deterministic truth, predictive enrichment, and generated phrasing.

\chapter{Context--Pattern Duality and Placeholder Generalisation}
\section{Observed Problem}
During the note-generation experiments, a recurring issue appeared: when the training material preserved too much lexical surface detail, the model tended to memorize examples instead of learning transferable pedagogical structure. This was especially problematic because ELA requires generalization to previously unseen vocabulary inside familiar grammatical patterns.

\section{Observed Improvement}
Performance improved after restructuring the target space around more abstract templates and placeholders. Instead of tying the generation task tightly to concrete lexical tokens, the revised setup emphasized structural roles and reusable pattern slots. The practical result was a stronger ability to produce pedagogical notes for unseen examples that shared grammatical structure with the training material.

\section{Empirical Path to the Claim}
The repository history shows that this idea did not appear as a finished theory from the start. It emerged through several practical turns. An early template-ID experiment on 2026-02-13 confirmed that deterministic template wiring was in place, but it also exposed a serious problem: after balancing, the dataset had only 511 rows, only 10 unique targets, and a duplicate ratio above 0.98, while both the refreshed baseline and the template-ID model produced a fallback rate of 1.0 on the small regression probe. In other words, the first attempt to constrain the target space became too collapsed to be useful.

This failure is important because it shows what the project actually learned. Placeholder-based abstraction helped only when it expanded transferable structure without collapsing target diversity. The later strategy documents therefore move away from ``T5 predicts the pedagogical truth'' and toward a classifier-first architecture in which compact structural prompts predict a closed template space and the final note is rendered from slots. This is a narrower but much more defensible use of abstraction.

The later artifacts also show why the interpretation remained valuable after that failed intermediate stage. By March 2026, the pipeline documentation and contract field-ownership rules had converged on the same separation: the contract and blueprints carry structural truth, while T5 is allowed to realize wording only. A phase-1 execution report dated 2026-02-25 confirms that this controlled rewrite path preserved classifier-owned blueprint fields while still producing user-facing generated notes. That is a much stronger foundation for the current thesis claim than raw seq2seq optimism would have been.

\section{Interpretive Framework}
This thesis interprets that behavior through a context--pattern duality:
\begin{itemize}
  \item during training, many contextual examples shape the internal pattern space of the model;
  \item during inference, the now-frozen patterns interpret new contextual instances.
\end{itemize}

In simple terms, context forms pattern in the open training phase, while pattern forms context in the closed inference phase. This is not presented here as a formal theorem. It is an empirical observation arising from the project's modelling path and used as an explanatory framework for why placeholder abstraction improved generalization.

\section{Lexical Invariance}
The key idea behind the placeholder strategy is lexical invariance. If the model learns to associate structural markers with pedagogical interpretation, then the same pattern can be reused when the surrounding words change. This is especially valuable in language learning, where the educational meaning of a structure often outlives any single example sentence.

The measured artifacts support a bounded version of this claim. Sample inference reports from the later hybrid book-enriched note path show exact sentence-level matches for several structurally distinct cases, including second conditionals, do-support negation, passive-plus-perfect constructions, existential agreement, and yes/no questions with auxiliary support. At the same time, phrase-level examples in the same artifact still expose unresolved slot-surface issues, because placeholder tokens such as \texttt{PART\_OF\_SPEECH} and \texttt{GRAMMATICAL\_ROLE} were emitted too literally. This matters for the thesis argument: the project did not discover perfect abstraction, but it did discover that lexical invariance becomes useful when the right information is abstracted and the remaining wording is tightly controlled.

\section{Scope of the Claim}
The claim should remain bounded. This thesis does not prove a universal law of learning systems, nor does it attempt to derive a physics result from model training behavior. The contribution is narrower and more defensible:
\begin{itemize}
  \item the project observed a generalization gain after switching from raw lexical notation to structurally abstract templates;
  \item the context--pattern duality provides a coherent explanation of that gain;
  \item the interpretation is useful for designing pedagogical generation tasks in ELA and similar systems.
\end{itemize}

  \item synchronization of documentation with implemented behavior.

Whether one accepts the phrase ``production-ready baseline'' in full or not, the report is useful because it captures the intended direction of the implementation phase: less noise, more traceability, and stronger safety around the contract itself.

\section{Deduplication and Quality Control}
A recurring theme in the project documentation is that raw volume is not enough. The system places substantial effort into:
\begin{itemize}
  \item filtering and capping repeated targets;
  \item excluding low-quality or telemetry-like note text;
  \item preserving CEFR balance and grammar-class coverage;
  \item validating note suitability and trace fields;
  \item reducing datasets to examples that remain useful after strict normalization.
\end{itemize}

The practical effect of this policy is visible in the processed dataset statistics. One hybrid T5 contract-target dataset built from the projected corpus started with 2,510 rows and ended with only 380 rows after deduplication and balancing, with 2,130 rows skipped by target-cap controls alone. This is a strong sign that the project prioritized controllable target diversity over raw size. It also explains why some intermediate model results remained weak: the engineering decision to compress and normalize the target space trades scale for discipline.

\section{Placeholder Construction}
The note-generation path appears to depend critically on converting enriched linguistic structures into more abstract templates. This is the point where dataset engineering and the context--pattern duality meet: the abstraction strategy is not only a training trick, but part of how the project reframes the pedagogical signal.

The canonical process document dated 2026-03-21 makes that transformation explicit. It defines a ten-step path from book-derived \texttt{notation + content} pairs to inference-time rendering:
\begin{enumerate}
  \item extract notation and paired content from source material;
  \item build a contract tree from the content;
  \item analyze the notation and convert it into placeholder-based template text;
  \item constrain placeholders to attributes available in the paired contract;
  \item train on templated notation rather than on fully rendered final notes;
  \item at inference time, rebuild the contract for new content;
  \item let the model output a notation template with allowed placeholders only;
  \item fill those placeholders from the contract;
  \item write the rendered note back into the tree.
\end{enumerate}

This is a stronger and more specific claim than ``the system uses placeholders.'' It shows that placeholders are intended to sit at the centre of the full data path from source extraction to runtime note realization.

The later slot-normalized projection artifacts make this even more concrete. In the v17 slot-normalized projection report, the repository records 3,505 projected rows, including 1,984 sentence-level slot-templated rows and 6,584 phrase-level slot-templated rows, with additional lexicalized-note repairs recorded explicitly. This demonstrates that placeholder construction was not only a conceptual strategy; it was operationalized as a measurable transformation stage over a large projected corpus.

\section{Template Opportunity Analysis}
The repository also contains reports that measure not just what was extracted, but what remained uncovered by the current template inventory. A template-opportunities report over the slot-normalized projection records 513 exact sentence families in projection, of which 205 had book notes and 202 already had book slot templates. The same report identifies phrase families with much weaker template coverage, especially around relative-clause-heavy phrase patterns. This is important because it shows that dataset engineering in ELA included gap analysis, not only corpus growth. The project did not simply build a dataset and train on it; it also measured where the current template space failed to cover the observed grammar families.

\section{Measured Template Space}
The classification-oriented template datasets show that even the template inventory itself was treated as a measurable object. One sentence-multilabel template dataset built from the covered projection reports 1,774 rows after filtering, 48 total template identifiers, 42 sentence template types, and 6 phrase template types. These counts matter because they convert the abstract idea of ``template diversity'' into something inspectable. They also help explain why some model and coverage decisions in later chapters could be made with more confidence: the project had begun to quantify the shape of its pedagogical pattern space rather than relying on intuition alone.

\section{From Free-Form T5 to Controlled Rendering}
The project documentation shows a clear evolution away from unconstrained note generation. By March 2026, the preferred architecture was no longer ``raw text to T5 note.'' The intended flow had become:
\begin{enumerate}
  \item validated linguistic contract;
  \item deterministic structural enrichment;
  \item CEFR prediction and blueprint generation;
  \item contract-to-template transformation;
  \item controlled lexical realization from that template;
  \item post-render validation that classifier-owned and contract-owned fields remain unchanged.
\end{enumerate}

This evolution is important for the thesis because it supports a more mature claim than ``the system uses T5.'' The stronger claim is that generation is being progressively confined to the narrowest part of the problem where wording variation is desirable but structural drift is unacceptable.

\section{Engineering Trade-off}
The dataset story also shows a cost of rigor. As examples become deduplicated, normalized, and filtered, the final usable set becomes much smaller than the initial raw corpus. That is not a weakness by itself; it is evidence that the project values controlled signal over inflated raw counts. It does, however, place a ceiling on the strength of the current empirical claims.

It identifies concrete failure classes that unit tests alone may miss: browser-side model availability, service-worker cache drift, ffmpeg-related rendering failures, missing proxy headers, and user-flow stalls during long-running media processing. For the thesis, this matters because it broadens the meaning of ``system validation.'' In ELA, validation is not only about linguistic correctness; it is also about whether a learner can move from upload to usable artifact through a stable interface path.

\chapter{Discussion, Limitations, and Future Work}
\section{Discussion}
The implemented ELA system supports the central thesis direction: authentic language can be made more accessible through layered structural explanation rather than through simplification. The combination of hierarchical representation, deterministic truth layers, and learner-facing visualization remains the strongest and most coherent aspect of the project.

The most interesting deeper result is that abstraction appears to matter more than raw model complexity in pedagogical generation. The placeholder-based route suggests that the project achieved its clearest modelling gains not by escalating model size, but by refining representational form. In practical terms, the project moved from asking a model to memorize many surface realizations toward asking it to learn a more stable mapping between structural context and explanatory pattern.

This is also where the engineering and research layers of the thesis meet most clearly. The same design instinct that pushed the runtime toward stricter contracts and deterministic boundaries also pushed the note-generation path toward canonicalized training signals. In both cases, the project became stronger when implicit variation was converted into explicit structure.

\section{Limitations}
The current limitations are both scientific and engineering-related:
\begin{itemize}
  \item some reported metrics still require careful reconciliation with the precise artifacts that produced them;
  \item the current branch contains unresolved regressions in the core pipeline;
  \item multimodal support is more fragile than text-only processing;
  \item some claims inherited from earlier project narratives need weaker wording in light of the current implementation state;
  \item the project still lacks a full user-study layer capable of demonstrating educational impact beyond technical plausibility.
\end{itemize}

\section{Future Work}
The most immediate future work falls into three clusters:
\begin{enumerate}
  \item \textbf{Stabilization.} Resolve current regressions in skeleton construction, translation projection, runtime validation boundaries, and media sentence preparation.
  \item \textbf{Model path maturation.} Continue the abstraction-based note-generation line, expand dataset evidence, and strengthen the empirical basis of the context--pattern duality claim.
  \item \textbf{Product hardening.} Improve media playback, subtitle synchronization, packaging, deployment consistency, and frontend/runtime ergonomics.
\end{enumerate}

Beyond those immediate priorities, the documentation also points toward longer-horizon directions: tighter offline packaging of runtime models, richer translation quality gates, broader corpus growth, stronger observability around pipeline failures, and more disciplined separation between classification, controlled rendering, and user-facing editing workflows.

\chapter{Conclusion}
This thesis has presented ELA as the continuation of an earlier research programme into explainable support for authentic English input. The project has evolved from a concept and prototype direction into a substantial full-stack implementation built around the RLE contract, a rules-first linguistic pipeline, multimodal processing, and a learner-facing frontend.

The core technical lesson of the work is twofold. First, explainability requires architecture, not just model output: contracts, validators, runtime boundaries, and structural representations matter. Second, abstraction in the training signal appears to be a decisive factor in pedagogical note generation, motivating the context--pattern duality as the main interpretive research contribution of this phase.

The project is not finished. Its current codebase still exposes unresolved regressions and integration mismatches, especially in the runtime and multimodal paths. Yet that incompleteness does not weaken the value of the thesis; it clarifies it. ELA is best understood not as a polished consumer product, but as a serious research-engineering system whose value lies in what has already been made explicit: a reusable architecture, a contract-centered representation model, a credible multimodal workflow, and a modelling direction that can be tested further rather than merely asserted.

In that sense, the continuation from the earlier research thesis is successful. The original pedagogical vision was not abandoned during implementation; it was forced into concrete interfaces, validators, services, datasets, and failure cases. That transition from concept to accountable system is the real achievement documented in this report.
